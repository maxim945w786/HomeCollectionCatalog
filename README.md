# HomeCollectionCatalog

## Описание проекта

**HomeCollectionCatalog** — учебная реляционная база данных для информационной системы учета домашней коллекции.

## Предметная область

Система предназначена для каталогизации личных коллекционных предметов: монет, марок, книг, виниловых пластинок, предметов искусства и других объектов. База данных позволяет хранить сведения о категориях коллекции, местах хранения, состоянии предметов, истории приобретения, оценке рыночной стоимости, страховых документах, планируемых покупках и цифровых экспозициях.

## Основные сущности

- `CollectionItems` — основная таблица коллекционных предметов.
- `Categories` — категории коллекции.
- `ItemTypes` — типы предметов.
- `StorageLocations` — места хранения.
- `AcquisitionSources` — источники приобретения.
- `AcquisitionHistory` — история приобретения.
- `Valuations` — оценка рыночной стоимости.
- `InsuranceDocuments` — страховые документы.
- `PurchasePlans` — планируемые покупки.
- `Exhibitions` и `ExhibitionItems` — цифровые экспозиции.
- `Roles` и `Users` — роли и пользователи системы.

## Роли пользователей

- **Пользователь** — просмотр коллекции.
- **Менеджер** — ведение каталога и планирование покупок.
- **Администратор** — полный доступ к данным и управлению пользователями.

## Тестовые данные

Скрипт `database.sql` создает базу данных `HomeCollectionCatalogDB` и заполняет ее тестовыми данными:

- 100 записей в основной таблице `CollectionItems`;
- 20 записей в справочнике `Categories`;
- 20 записей в справочнике `ItemTypes`;
- 20 записей в справочнике `StorageLocations`;
- 20 записей в справочнике `AcquisitionSources`;
- дополнительные данные для истории приобретения, оценки, страхования, планов покупок и экспозиций.

## Используемая СУБД

MS SQL Server.

## Как запустить

1. Открыть SQL Server Management Studio.
2. Создать новый запрос.
3. Вставить содержимое файла `database.sql`.
4. Выполнить скрипт.
5. Проверить созданную базу данных `HomeCollectionCatalogDB`.

## Изменения в практической работе №15

В структуру базы данных добавлено поле:

```sql
Image VARBINARY(MAX)
```

Поле предназначено для хранения изображений коллекционных предметов в бинарном формате (BLOB).

Для тестирования все записи основной таблицы CollectionItems были заполнены тестовыми бинарными данными.

Доработанный SQL-скрипт сохранён в файле `database.sql`.
